# XMU 电子设计竞赛智能小车项目

本仓库整理自 XMU 电子设计竞赛智能小车项目，项目最终获得 **XMU 电子设计竞赛二等奖**。项目方向与 2024 年 H 题智能小车方向接近，主要围绕小车硬件搭建、传感器检测、运动控制、循迹和任务执行展开。

当前仓库代码主要来自 `ex7-1 mpu6050` 派生工程。工程保留了老师提供或参考的 MPU6050/DMP、STM32 标准外设库、编码器/电机等基础代码，并在此基础上扩展了任务状态机、五路灰度循迹、左右轮速度控制、MPU6050 yaw 纠偏、OLED 计时、蜂鸣器/LED 提示和 VOFA 调试输出。

> 证据边界：XMU 电子设计竞赛二等奖、整车成本约 16 元、赛前通宵调试、个人承担主要推进角色等信息来自项目参与者补充，未在源码文件中直接体现。源码和工程文件能直接体现的是硬件接口、模块接入、控制逻辑、任务结构和调试输出。

## 项目背景

项目面向 XMU 电子设计竞赛智能小车任务，目标是在低成本硬件条件下完成自动行驶、循迹、关键点提示和时间显示。根据当前文件夹中的任务书 PDF，直接体现的题目材料是校赛低年级组《自动行驶小车》：小车需要在场地内沿黑色线迹和弧线完成 A、B、C、D 点相关路线，并在关键点或终点进行声光提示，同时显示行驶时间。

结合项目参与者补充，本项目的突出特点是低成本实现。小车总成本约 16 元，硬件资源和传感器稳定性都比较有限，因此项目难点不只是写出控制代码，而是在低成本、低稳定性硬件条件下，通过反复实车调试让系统实际跑起来。

## 个人工作与项目特点

项目并非完全从零独立完成，工程中能看到老师提供或参考的 6050 基础工程、MPU6050/DMP 驱动和部分控制相关代码。但在项目落地过程中，我承担了主要推进角色，工作覆盖硬件搭建、模块连接、灰度传感器和 6050 模块调试、运动控制逻辑调整、实车测试和赛前问题排查。

项目中的典型问题包括灰度传感器识别不稳定、左右轮速度一致性不足、直线行驶容易跑偏、速度环和方向控制需要反复调参等。赛前进行了长时间高强度调试，基本通宵排查到第二天上午，持续解决现场暴露的问题。该赛前调试过程来自项目参与者补充，未在文件中直接体现。

## 任务实现

当前最完整工程位于：

```text
26年五月电赛校赛/car-deepseek/car-deepseek/ex7-1 mpu6050
```

`user/main.c` 通过 PB1/PB9 的短按和长按选择四个任务，`user/task.c` 中对应实现了：

- `Task1`：A 到 B 直线行驶，距离目标约 100 cm；
- `Task2`：混合一圈路线；
- `Task3`：A-C-B-D-A 一圈路线；
- `Task4`：A-C-B-D-A 四圈路线。

## 硬件与接口

以下内容来自当前工程文件和源码：

- 主控：Keil 工程 `user/ex2.uvprojx` 目标器件为 `STM32F103C8`；
- 姿态模块：`driver/mpu6050/` 包含 `MPU6050.c`、`IOI2C.c`、`inv_mpu.c`、`inv_mpu_dmp_motion_driver.c`、`filter.c`，`main.c` 中通过 DMP 更新 `Yaw`；
- 灰度/循迹传感器：`user/bsp_track.c` 读取 5 路输入，管脚为 PC9、PB12、PA11、PC10、PC12；
- 电机驱动：`user/bsp_motor.c` 使用 TIM2 PWM，PA0/PA1 为 PWM 输出，PA5/PA4/PB4/PB3 为方向控制；
- 编码器：`user/bsp_encoder.c` 使用 TIM3、TIM4 编码器接口，PA6/PA7 与 PB6/PB7 输入，并按轮径、减速比换算速度和里程；
- 提示与显示：`user/bsp_buzzer.c`、`user/bsp_led.c`、`user/bsp_oled.c` 提供蜂鸣器、LED 和 OLED 时间显示；
- 调试输出：`driver/usart1.c` 中的 `vofa_send_multi()` 用于输出姿态、目标速度、轮速、里程等调试数据。

## 软件结构

- `user/main.c`：系统初始化、按键任务选择、SysTick 1 ms 时基、10 ms 控制周期、VOFA 调试输出；
- `user/task.c` / `user/task.h`：Task1-Task4 任务状态机、直线/弧线段切换、路线距离阈值、声光提示和运行计时；
- `user/pid_control.c`：左右轮速度 PID、积分限幅、输出限幅和目标速度渐变；
- `user/track_control.c`：基于循迹偏差的方向修正；
- `user/bsp_track.c`：五路灰度传感器读取和偏差计算，未识别到线迹时返回 `88`；
- `user/bsp_encoder.c`：编码器测速、里程累计和距离清零；
- `user/bsp_motor.c`：PWM 与方向控制；
- `driver/mpu6050/`：MPU6050/DMP 基础驱动；
- `cmsis/`、`st_lib/`：STM32F10x CMSIS 与标准外设库。

## 控制逻辑

程序启动后完成 LED、蜂鸣器、电机、编码器、五路灰度传感器、速度 PID、循迹控制、MPU6050/DMP、OLED、按键和串口初始化。`SysTick` 设置为 1 ms 中断，每 10 ms 更新编码器速度、读取 DMP 姿态并执行当前任务控制；运行时每 100 ms 通过 VOFA 输出关键变量。

直线段主要使用 MPU6050 yaw 作为方向反馈，结合编码器里程判断终点和减速区间。弧线或循迹段主要使用五路灰度传感器偏差，经 `track_control.c` 计算左右轮速度差。`task.c` 中还包含丢线处理逻辑，短时间未识别到线时使用较低搜索速度和上一次偏差尝试恢复。

## 典型问题与处理

- 灰度传感器成本低、稳定性较差：代码中使用五路传感器加权偏差，并用 `88` 表示未识别到线迹，在任务逻辑中加入丢线搜索和超时停止；
- 左右轮速度一致性不足：通过 TIM3/TIM4 编码器读取左右轮速度，并在 `pid_control.c` 中分别做速度 PID；
- 直线行驶容易跑偏：直线段叠加 MPU6050 yaw 纠偏，使用 `target_yaw_task1`、`yaw_error_straight` 和 `yaw_correction` 调整左右轮目标速度；
- 调参依赖实车反馈：VOFA 输出 yaw、目标速度、轮速、里程等数据，用于观察和调整控制参数；
- 赛前高强度调试：该过程来自项目参与者补充，未在文件中直接体现。

## 编译与烧录

1. 使用 Keil MDK 打开 `user/ex2.uvprojx`。
2. 确认目标器件为 `STM32F103C8`。
3. 确认包含路径中有 `..\st_lib\inc;..\cmsis;..\user;..\driver;..\driver\mpu6050`。
4. Rebuild 工程后，通过 ST-Link 或对应下载器烧录到目标板。
