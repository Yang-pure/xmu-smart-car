# XMU 电子设计竞赛智能小车项目

这是我参加 XMU 电子设计竞赛时做的智能小车项目，最终获得 **XMU 电子设计竞赛二等奖**。项目方向与 2024 年 H 题智能小车方向接近，主要围绕小车硬件搭建、传感器检测、运动控制、循迹和任务执行展开。

> 本仓库同时用于记录 Git 基础操作与项目版本管理实践。

这个项目对我来说比较有代表性的一点是：它不是单纯把代码写出来，而是在很低成本、硬件稳定性一般的条件下，把小车一步步调到能稳定完成比赛任务。我们的小车总成本大约 16 元，和其他队伍几百元的方案相比，硬件资源非常有限，所以很多问题都需要通过实车测试不断暴露、定位和修正。

项目并不是完全从零独立完成，前期使用了老师提供或参考的 `ex7-1 mpu6050` 工程，以及部分 MPU6050/DMP、控制相关基础代码。但后续从硬件组装、传感器接入、控制逻辑调试到最终实车运行，我在项目落地过程中承担了主要推进角色，尤其是赛前调试和问题排查部分投入比较多。

## 我主要做了什么

- 搭建并调试小车硬件，包括电机驱动、编码器、灰度传感器、MPU6050、OLED、蜂鸣器和 LED 等模块；
- 在老师 6050 基础工程上继续配置和扩展小车工程，把传感器、控制和任务执行逻辑整合到同一个 Keil 工程中；
- 调试左右轮速度闭环、MPU6050 yaw 直线纠偏、五路灰度循迹和任务状态机；
- 针对直线跑偏、左右轮速度不一致、灰度识别不稳定、丢线等问题进行实车排查；
- 通过 VOFA 输出 yaw、轮速、目标速度、里程等数据，配合实车测试反复调参；
- 赛前进行了长时间高强度调试，基本通宵排查到第二天上午，最终完成比赛任务。

## 工程说明

当前整理上传的是最终较完整的一版 6050 派生工程：

```text
26年五月电赛校赛/car-deepseek/car-deepseek/ex7-1 mpu6050
```

工程目标芯片为 `STM32F103C8`，使用 Keil MDK 打开：

```text
user/ex2.uvprojx
```

主要目录：

- `user/`：小车应用层代码和 Keil 工程文件；
- `driver/mpu6050/`：MPU6050、DMP、软件 I2C 和滤波相关基础驱动；
- `driver/`：USART、NVIC、控制相关基础文件；
- `cmsis/`：CMSIS 启动和内核文件；
- `st_lib/`：STM32F10x 标准外设库；
- `PROJECT_EXPERIENCE_SUMMARY.md`：更正式的项目经历总结，可用于简历、综测或面试准备。

## 硬件与接口

代码中可以直接看到的主要模块和接口如下：

- 主控：`STM32F103C8T6`；
- 姿态模块：MPU6050/DMP，工程中包含 `MPU6050.c`、`IOI2C.c`、`inv_mpu.c`、`inv_mpu_dmp_motion_driver.c`、`filter.c`；
- 灰度/循迹传感器：五路输入，`bsp_track.c` 中使用 PC9、PB12、PA11、PC10、PC12；
- 电机驱动：TIM2 PWM，PA0/PA1 为 PWM 输出，PA5/PA4/PB4/PB3 为方向控制；
- 编码器：TIM3/TIM4 编码器接口，PA6/PA7 和 PB6/PB7 输入；
- 显示与提示：OLED、蜂鸣器、LED；
- 调试输出：USART1 + VOFA。

## 软件实现

### 任务选择

`main.c` 中通过 PB1/PB9 的短按和长按选择不同任务：

- PB1 短按：`Task1`，A 到 B 直线行驶，距离约 100 cm；
- PB9 短按：`Task2`，混合一圈路线；
- PB1 长按：`Task3`，A-C-B-D-A 一圈路线；
- PB9 长按：`Task4`，A-C-B-D-A 四圈路线。

### 控制周期

程序使用 `SysTick` 作为 1 ms 时基，每 10 ms 更新编码器速度、读取 DMP 姿态并执行当前任务控制；运行时每 100 ms 通过 VOFA 输出关键变量，方便观察小车状态和调参。

### 直线控制

直线段主要依赖 MPU6050 的 yaw 角进行方向反馈。任务初始化时记录目标 yaw，行驶过程中计算 `Yaw - target_yaw_task1`，再通过 `yaw_correction` 调整左右轮目标速度，尽量减小直线跑偏。

### 循迹控制

`bsp_track.c` 读取五路灰度传感器并计算偏差，未识别到线迹时返回 `88`。`track_control.c` 根据偏差计算左右轮速度差，用于弧线和循迹段控制。`task.c` 中也加入了丢线后的低速搜索和异常停止逻辑。

### 速度闭环

`bsp_encoder.c` 通过 TIM3/TIM4 读取左右轮编码器脉冲，并换算速度和里程。`pid_control.c` 对左右轮分别进行速度 PID 控制，加入积分限幅、输出限幅和目标速度渐变，用来缓解轮速不一致和启动突变问题。

## 遇到的问题

这个项目中最费时间的部分不是写某一个函数，而是不断把代码和实车状态对起来。比较典型的问题包括：

- 灰度传感器成本低，识别不够稳定；
- 左右轮速度一致性不足，直线容易偏；
- 速度环、方向控制和灰度反馈之间需要反复调参；
- 低成本硬件误差比较大，很多问题只有跑起来才会暴露；
- 比赛前时间紧，需要快速定位问题并完成能稳定运行的版本。

对应的处理思路主要是：用编码器做速度闭环，用 MPU6050 yaw 做直线纠偏，用五路灰度传感器做循迹偏差判断，再通过 VOFA 数据和实车表现反复调整参数。

## 编译与烧录

1. 使用 Keil MDK 打开 `user/ex2.uvprojx`。
2. 确认目标芯片为 `STM32F103C8T6`。
3. 确认包含路径中有：

```text
..\st_lib\inc;..\cmsis;..\user;..\driver;..\driver\mpu6050
```

4. Rebuild 工程后，通过 ST-Link 或对应下载器烧录到目标板。

Keil 生成的 `Objects/`、`Listings/`、`DebugConfig/`，以及 `.uvoptx`、`.uvguix.*` 等本地配置文件没有上传，已经通过 `.gitignore` 排除。
